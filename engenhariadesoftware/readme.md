O trecho indicado trata manejo do código, onde escrever o código não é o único passo que o mantém, é importante lembrar que o código precisa passar por manutenção e organização constante para que siga atingindo seu objetivo. Utilizar um rigor maior em boas práticas pode ser uma boa maneira, segundo o texto, de mantê-lo adaptado e facilitar possíveis mudanças que venham a ser necessárias. Ele coloca um ciclo que vai de concepção, introdução, manutenção e depreciação, e abre debate para "que praticas podem ser utilizadas para melhoria do produto". São apontados três princípios utilizados no design, arquitetura e escrever códigos, são eles: tempo e mudança, para adaptação do código; escala e crescimento, como facilitar uma organização para adaptação e evolução; trade-offs e custos, como tomar decisões com objetivo de melhorar e apoiar os outros princípios.
